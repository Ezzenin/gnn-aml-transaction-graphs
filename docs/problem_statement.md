# Постановка задачи

## Предметная область и актуальность
Отмывание денег (money laundering) — легализация доходов, полученных преступным
путём; по оценке УНП ООН, ежегодно отмывается порядка **2–5% мирового ВВП**
(Altman et al., 2023). Противодействие (AML / антифрод) опирается на выявление не
отдельных «плохих» транзакций, а **цепочек операций** — типовых схем, которыми
средства проводят через сеть счетов (дробление, транзит, круговые переводы) на
стадии layering. Естественная модель таких данных — **направленный мультиграф**:
вершины — счета, рёбра — транзакции (между парой счетов их может быть много).
Цепочки — это локальные подграфы характерной топологии (8 паттернов ниже), поэтому
задача по природе **графовая**: одиночная транзакция вне контекста соседства
неотличима от легитимной, а «подозрительность» проявляется в структуре окрестности.
Отсюда интерес к графовым нейросетям (GNN) и их сравнение с табличными моделями и
классическими графовыми эвристиками. Практическая значимость — рост объёмов
безналичных платежей и требования регуляторов к объяснимым системам мониторинга.

## Цель
Исследовать, как способ построения графового представления транзакционных данных
и выбор архитектуры GNN влияют на качество выявления цепочек (паттернов) финансовых
операций; сравнить с неграфовыми (XGBoost) и эвристическими альтернативами;
продемонстрировать результат в виде мини-инструмента (режим антифрод).

## Исследовательские вопросы
1. Представление графа: «узел = счёт, ребро = транзакция» (edge-classification)
   против «узел = транзакция» (node-classification) — что лучше для детекции цепочек?
2. Вклад мультиграфовых адаптаций (направленность, reverse message passing,
   временной порядок рёбер) поверх базовых GNN.
3. Граница применимости: на каких паттернах GNN уверенно превосходит XGBoost
   и классические эвристики (перечисление циклов, подсчёт мотивов)?

## Формальное определение «цепочек» = 8 паттернов AMLSim
В направленном мультиграфе транзакций:
1. **fan-out** — из одной вершины рёбра к ≥2 вершинам;
2. **fan-in** — в одну вершину рёбра от ≥2 вершин;
3. **gather-scatter** — fan-in и fan-out одной вершины;
4. **scatter-gather** — fan-out вершины и fan-in другой через общий набор промежуточных;
5. **simple cycle** — деньги возвращаются в исходную вершину;
6. **random** — как цикл, но средства не возвращаются;
7. **bipartite** — перевод из набора входных счетов в набор выходных;
8. **stack** — двудольный паттерн с дополнительным слоем.

Стадии отмывания: placement → layering → integration; паттерны относятся к layering.

## Решение по scope (фактически реализовано)
- **Ядро (research, выполнено):** антифрод / AML — Elliptic (Этап 1) и IBM AML
  HI-Small (Этап 2); бейзлайны (XGBoost), edge-GNN + ablation мультиграфовых
  адаптаций, классические эвристики; метрики AUC-PR / F1-minority / recall@P,
  разбивка по 8 паттернам; сравнение с литературой.
- **Продукт (выполнено):** Streamlit-режим «антифрод» — скоринг транзакций
  обученной edge-GNN + визуализация ego-подграфа цепочки.
- **В перспективах (НЕ реализовано):** RQ1 в части «узел = транзакция» (line-graph)
  — сделана только постановка edge-classification «узел=счёт»; режим Personal
  finance (follow-the-money на синтетике); темпоральные модели (EvolveGCN);
  edge-updates / раздельные агрегаторы для приближения к SOTA Egressy.

## Статус ответов на RQ (кратко)
- **RQ1** — закрыт частично: реализована и измерена edge-classification («узел=счёт»);
  альтернатива «узел=транзакция» вынесена в перспективы.
- **RQ2** — закрыт: ablation адаптаций reverse/port/ego в трёх режимах; в нашем
  (ослабленном) режиме прироста над базовой GINe нет — границы переноса вывода
  Egressy очерчены (см. `lit_review.md`, `lit_benchmarks.md`).
- **RQ3** — закрыт: F1 по 8 паттернам для XGBoost / GNN / эвристик; XGBoost лидирует
  везде, степенные эвристики неинформативны (illicit-счета низкостепенные).

## Метрики
Главные: **AUC-PR**, **F1 по позитивному (illicit) классу**. Практическая:
**recall @ fixed precision**. Вторичная: ROC-AUC. Для цепочек: F1 по каждому
из 8 паттернов. Accuracy — не отчётная метрика (дисбаланс ~2%).
