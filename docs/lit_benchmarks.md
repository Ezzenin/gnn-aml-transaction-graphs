# Бенчмарки из литературы — опорные числа для сравнения (Этап 2)

> Назначение: единый справочник опубликованных результатов на **тех же датасетах**
> (IBM AML HI-Small и Elliptic), с которыми сравниваются наши `results/*`.
> Используется в коде/отчёте при сборке сравнительных таблиц и в анализе расхождений.
> Все числа перепроверены по первоисточникам (ссылки внизу). Метрика везде, где не
> указано иное, — **F1 по миноритарному (illicit) классу**, в процентах.

## 1. IBM AML (AMLworld) — edge-classification, наш основной бенчмарк

### 1.1. Egressy et al., AAAI 2024 (Multi-GNN) — ключевая таблица для RQ2

Сетап авторов: temporal split **60/20/20** по timestamp'ам, neighborhood sampling
(Hamilton et al.) для всех GNN, метрика — minority-class F1, среднее по 5 запускам.
«AML Small HI» = наш HI-Small (illicit ≈ 0.1%).

Ключевые выводы авторов (важно для интерпретации нашего ablation):
- На Small HI адаптации поднимают GIN с 28.7% до 57.2% (≈ +30 п.п.); **наибольший
  вклад — reverse MP и port numbering** (28.7 → 56.9); **ego-IDs почти не добавляют**
  поверх rev+port (но дают заметный прирост поодиночке: 28.7 → 39.65).
- Авторы сами оговаривают: использовалось только 2 GNN-слоя, с бóльшей глубиной
  вывод про ego-IDs может измениться (длинные циклы).
- **GIN+EU (edge updates) — самый дешёвый сильный буст** (28.7 → 47.7 без
  каких-либо мультиграфовых адаптаций); полный Multi-GIN+EU (64.8) обходит GBT.
- Multi-PNA+EU — единственный вариант, обыгравший GBT+GF на **всех** AML-датасетах.
- Per-pattern (приложение к статье): **бо́льшая часть illicit-транзакций, входящих в
  паттерны, ловится**; итоговый F1 по датасету сильно тянут вниз «одиночные»
  illicit-транзакции вне паттернов (lone transactions) — прямой аналог нашей группы
  `unknown` (1968 из 5177 illicit без привязки к паттерну).

### 1.2. Сопоставление с нашими результатами (results/ibm_*.json) и причины разрыва

Наши числа (полный HI-Small, temporal split 65/15/20, порог по val): XGBoost
F1 = 0.190 / AUC-PR = 0.129; GINe base F1 = 0.134; +rev 0.078; +port 0.125;
+ego 0.151; full 0.066. У Egressy GIN base = 28.7, Multi-GIN = 57.2, XGB+GF = 63.2.

Числа **несопоставимы напрямую**, расхождение объяснимо и должно явно
оговариваться в отчёте:

1. **Сплит**: у нас 65/15/20 vs 60/20/20; доля illicit в нашем test выше train
   (0.18% vs 0.08%) — сдвиг распределения внутри датасета.
2. **Обучающая выборка**: мы обучаем на сабсэмпле (`neg_ratio: 20`, ~54k рёбер);
   Egressy обучают на **всех** train-рёбрах с весами классов. Это, вероятно,
   главный фактор: модель видит на 2 порядка меньше негативов.
3. **Ёмкость/окрестность**: у нас `num_neighbors [10,10]`, 2 слоя, hidden 64; в
   референсной реализации IBM/Multi-GNN окрестности и батчи существенно больше
   (точные гиперпараметры — в `github.com/IBM/Multi-GNN`, сверить перед финальным
   прогоном; в статье они в Appendix F).
4. **Edge updates**: у нас их нет; GIN → GIN+EU в статье даёт +19 п.п. — самый
   дешёвый кандидат на доработку, если останется время.
5. **GF-фичи у GBT**: их XGBoost+GF включает подграфовые фичи (fan/cycle/…) из
   GFP (см. 1.3); наш XGBoost использует только degree/суммы/параллельные рёбра —
   поэтому наш XGBoost (0.19) много ниже их 63.2.
6. **Наша аномалия с reverse MP** (просадка вместо прироста) после фикса
   (reverse до семплинга) остаётся фактом наших экспериментов; у Egressy reverse —
   главная адаптация. Кандидаты-объяснения для отчёта: малые окрестности [10,10]
   (двунаправленный граф удваивает конкуренцию за слоты семплера), отсутствие
   раздельных весов AGG_in/AGG_out (у нас один GINEConv на оба направления с
   флагом, у них — раздельные агрегаторы), сабсэмплинг негативов.

**Честная формулировка для отчёта:** мы воспроизводим *направление* эффекта
адаптаций в ослабленном вычислительном режиме и фиксируем, где выводы Egressy
не переносятся (reverse при малых окрестностях), а где переносятся (ego/port ≥ base).

### 1.3. Blanuša et al., ICAIF 2024 — Graph Feature Preprocessor (GFP)

Подграфовые фичи (fan-in/out, degree, scatter-gather, циклы, временные окна),
считаемые потоково, + GBT:
- Без графовых фич максимум LightGBM/XGBoost на AML **HI** ≈ **24.5%** F1
  (наш XGBoost 19.0% — согласуется по порядку);
- Уже добавление **только fan-in/fan-out фич даёт > +30 п.п.** F1;
- GFP+GBT достигает уровня лучших GNN при батчевой обработке в реальном времени.

Практическое следствие для нас: усиление XGBoost-бейзлайна fan-фичами в
временном окне — дешёвая и опубликованно-обоснованная доработка (Фаза E+/stretch).

### 1.4. Altman et al., NeurIPS 2023 (Datasets & Benchmarks) — сам датасет

- Источник HI-Small и разметки паттернов; оценка ООН: 2–5% мирового ВВП
  отмывается ежегодно.
- Бейзлайны статьи = строки GBT/GNN в таблице 1.1 (общие с Egressy).
- Наблюдение авторов: PNA почти на уровне GBT+GF **без ручных фич** — аргумент
  «за» выразительные GNN.
- Также показано: модели, обученные на объединённых данных нескольких банков,
  значительно сильнее одно-банковских (контекст для «Перспектив»).

### 1.5. Свежие работы на тех же датасетах (для раздела «Перспективы» и сравнения)

| Работа | Метод | Результат на AML | Статус |
|---|---|---|---|
| Bilgi, Chen, Atasu, 2024/2025 (arXiv:2412.00241, **MEGA-GNN**) | двухступенчатая агрегация: сначала параллельные рёбра, затем соседи; permutation-equivariant | **до +13 п.п.** к SOTA (Multi-GNN) на AML; в расширенной версии: в среднем +16.6 к base GNN, +5.0 к Multi-GNN, +2.9 к Multi-FraudGT | проверено по абстракту/препринту |
| Multi-FraudGT (граф-трансформер FraudGT + адаптации) | трансформер на рёбрах | упоминается как сильный бейзлайн в MEGA-GNN; чувствителен к перестановке портов | выходные данные FraudGT сверить перед цитированием |
| TeMP-TraG, 2025 (arXiv:2503.16901) | temporal message passing по рёбрам транзакций | улучшения на IBM AML | детали сверить |

Вывод для отчёта: линия «направленный мультиграф + edge-classification» активно
развивается (Multi-GNN → FraudGT → MEGA-GNN), и наш ablation воспроизводит её
базовый строительный блок.

## 2. Elliptic — node-classification (контекст Этапа 1)

### 2.1. Weber et al., 2019 (arXiv:1908.02591) — каноничный бейзлайн

- Сетап: train = шаги 1–34, test = 35–49 (temporal), illicit F1.
- **GCN: F1 = 0.628** (precision 0.812, recall 0.512); skip-GCN немного лучше.
- **RandomForest (все фичи AF): F1 ≈ 0.796** — деревья сильнее GNN.
- Независимое воспроизведение (Lorenz et al., 2020, arXiv:2005.14635):
  RF ≈ **0.83**, XGBoost ≈ **0.76**, LogReg ≈ **0.45** — наши Этап-1 числа
  (XGBoost F1 0.792 / AUC-PR 0.80; LogReg 0.379) попадают в этот коридор.
- **Dark market shutdown на шаге 43**: все модели резко проседают после него —
  опубликованное объяснение нашего val→test разрыва у GNN (наш порог/val — ранние
  шаги, test — поздние).

### 2.2. Pareja et al., AAAI 2020 (EvolveGCN)

- На Elliptic минорный F1: EvolveGCN-O > статический GCN, но GCN-GRU ещё лучше;
  точные числа даны графиком (не таблицей) — в отчёте ссылаться на ранжирование,
  не на конкретное значение.
- Подтверждают эффект шага 43: после shutdown падают **все** модели, статические —
  сильнее всего. Это опубликованная мотивация нашей F2 (EvolveGCN) и
  `temporal_val: true`.

### 2.3. Deprez et al., 2025 (INFORMS Journal on Data Science 5(2):119–154)

Систематический обзор (97 работ) + единый эксперимент на Elliptic:
- сетевые признаки/методы повышают качество; **в их сетапе GNN дают лучший
  результат на Elliptic**;
- но: осторожность с GNN при сильном дисбалансе и специфичной топологии;
  **синтетические данные могут давать чрезмерно оптимистичные оценки** —
  обязательная оговорка для нашего IBM-блока;
- открытый код: `github.com/B-Deprez/AML_Network` (можно сверять реализацию
  эгонет-фич и протокол).

### 2.4. Свежая переоценка GNN на Elliptic (для нашей дискуссии XGBoost > GNN)

Препринт 2025/26 (seed-matched сравнение inductive vs transductive протоколов;
выходные данные уточнить перед цитированием — найден по абстракту):
- при **строго индуктивном** протоколе RF на сырых фичах F1 = 0.821 обходит все
  GNN (лучший GraphSAGE = 0.689 ± 0.017);
- разрыв ~39.5 п.п. F1 объясняется **доступом к test-периодной топологии при
  обучении** (утечка в transductive-протоколе);
- ablation с перемешиванием рёбер: случайная топология работает не хуже реальной
  при временном сдвиге.

Это прямое опубликованное подтверждение нашего Этап-1 результата (XGBoost 0.80
AUC-PR > SAGE 0.66) — наш протокол как раз строгий (порог по val, фичи по train).

## 3. Чек-лист корректного сравнения (для compare.py и текста)

1. Мы отчитываем **AUC-PR + F1-minority + recall@P90**; литература по IBM —
   только F1-minority. В сравнительной таблице с литературой использовать F1.
2. Порог: у нас фиксируется по val — у Egressy аналогично (val-based selection);
   протоколы совместимы по духу, но доли сплитов разные — в таблице с чужими
   числами всегда давать сноску «другой сплит/режим обучения».
3. Никогда не ставить наши и чужие числа в одну колонку без пометки; формат:
   отдельный блок «Reference results (literature)» в `results/ibm_comparison.md`.
4. Группа `unknown` (illicit вне паттернов) — у Egressy это «lone transactions»,
   признанные самыми трудными; в per-pattern выводах разделять «паттерновые» vs
   «одиночные» позитивы.
5. Если будет время на доработки, приоритет по ожидаемому эффекту из литературы:
   (a) обучение на всех train-рёбрах с class weights вместо neg_ratio-сабсэмпла;
   (b) edge updates (GIN+EU, +19 п.п. у Egressy); (c) увеличение num_neighbors;
   (d) fan-in/out GF-фичи для XGBoost (+30 п.п. у GFP).
